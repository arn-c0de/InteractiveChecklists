# AH-64D Apache Navigation Checklist

## Pre-Flight Navigation Planning

- [ ] **Mission route** - Plan and brief with crew
  - Departure point, en route waypoints, target area, return
  - Total distance and estimated flight time
  - Fuel required vs available
- [ ] **Waypoints** - Prepare coordinates (MGRS or Lat/Long)
  - Home base/departure
  - Checkpoints (recommend every 5-10 km)
  - Target area waypoints
  - Battle positions (BP)
  - FARP (Forward Arming and Refueling Point)
  - Alternates
- [ ] **Threat areas** - Mark on map
  - Known AAA (Anti-Aircraft Artillery) positions
  - SAM (Surface-to-Air Missile) sites and engagement zones
  - Radar coverage areas
  - Plan stand-off distances or avoidance routes
- [ ] **Terrain analysis** - Study route
  - Identify masking terrain for concealment
  - Note obstacles (towers, wires, power lines)
  - Identify forced landing areas
- [ ] **Communication plan** - Brief frequencies and checkpoints
  - Tactical frequencies (FM1/FM2)
  - Tower/approach frequencies (VHF/UHF)
  - Checkpoints for position reports
- [ ] **Weather briefing** - Review conditions
  - Winds aloft (affects fuel and groundspeed)
  - Visibility, cloud ceiling
  - Temperature (affects hover performance)
- [ ] **NOTAMS** - Review active

## Navigation System Setup (Both Stations)

- [ ] **TSD page on MPD** - Access
  - Press TSD button on MPD bezel
- [ ] **GPS status** - Verify
  - Check satellite count (minimum 4, ideally 6+)
  - PDOP (Position Dilution of Precision) <4 ideal
  - GPS accuracy should show <10 meters
- [ ] **Present position** - Verify on TSD
  - Aircraft symbol should match known parking location
  - Compare TSD position with visual landmarks
- [ ] **Keyboard Unit (KU)** - Use to enter waypoints
  - Press POINT on KU
  - Enter waypoint identifier (e.g., WP01)
  - Enter coordinates (MGRS or Lat/Long format)
  - Store waypoint
- [ ] **Waypoints to enter:**
  - **H (Home)** - Home base coordinates
  - **C (Current)** - Current position (auto-updates)
  - **T (Target)** - Primary target area
  - **WP01-WP20** - Route waypoints as required
  - **F (FARP)** - Forward refuel/rearm point
  - **A (Alternate)** - Alternate landing site
  - **B1-B9** - Battle positions
- [ ] **Route building on TSD** - Create planned route
  - Press ROUTE on MPD
  - Select waypoints in sequence
  - Verify route displays on TSD
  - Check total route distance and bearing
- [ ] **Hazard points** - Mark threats on TSD
  - Press POINT, select HAZARD type
  - Enter threat location coordinates
  - Set threat type (SAM, AAA, etc.)
  - Set threat range (engagement zone radius)

## TSD Display Configuration

- [ ] **TSD page selected** - On both CPG and Pilot MPDs
- [ ] **Map scale** - Adjust for phase of flight
  - Tactical planning: 25 km or 50 km
  - En route cruise: 10 km or 25 km
  - Terminal/target area: 5 km or 2 km
  - Nap-of-earth: 1 km or 0.5 km
  - Use TSD zoom buttons (+/-)
- [ ] **Map orientation** - Select
  - NORTH UP: Map fixed with north at top (easier for planning)
  - TRACK UP: Map rotates, track always up (easier for flying)
  - Press ORIENT button on TSD page to toggle
- [ ] **TSD symbology** - Configure display
  - Press SHOW button to access filters
  - Aircraft position symbol - ON
  - Waypoint symbols - ON
  - Route line - ON
  - Threats/hazards - ON
  - Phase lines - As required
  - Control measures - As required
- [ ] **Aircraft position icon** - Verify centered or visible
- [ ] **Waypoint data box** - Check displays bearing/range to next waypoint
- [ ] **HSI mode on FLT page** - Set to GPS for navigation

## Navigation Instruments Check (Pilot Station)

- [ ] **FLT page on MPD** - Select and verify
  - HSI (Horizontal Situation Indicator) - Check displayed
  - Altimeter - Verify QNH set correctly
  - Airspeed indicator - Operational
  - Vertical speed indicator - Operational
- [ ] **HSI navigation source** - Set to GPS
  - Course arrow points to active waypoint
  - Bearing and range to waypoint displayed
- [ ] **Heading** - Cross-check
  - Compare magnetic heading on HSI with standby compass
  - Verify GPS track matches heading (if no crosswind)
- [ ] **Altitude sources** - Check both
  - Barometric altitude (MSL - Mean Sea Level)
  - Radar altimeter (AGL - Above Ground Level)
- [ ] **GPS status on TSD** - Verify 4+ satellites, PDOP <4

## En Route Cruise Navigation

- [ ] **Cruise airspeed** - Establish and maintain
  - Normal cruise: 120-140 KIAS
  - Fast cruise: 140-160 KIAS (reduced endurance)
  - Economy cruise: 100-120 KIAS (max endurance)
- [ ] **Cruise altitude** - Establish based on mission
  - High cruise: 500-1,500 ft AGL (low threat, good visibility)
  - Medium altitude: 200-500 ft AGL (moderate threat)
  - Low level: 50-200 ft AGL (high threat environment)
- [ ] **TSD monitoring** - Continuous
  - Aircraft position on planned route
  - Distance to next waypoint decreasing
  - Bearing to waypoint matches HSI
- [ ] **Waypoint sequencing** - Monitor auto-sequence
  - As waypoint is reached, next waypoint becomes active
  - Verify correct waypoint sequenced
  - Manual sequence available if needed (press WP button on TSD)
- [ ] **Fuel monitoring (EUFD FUEL page)** - Check
  - Current fuel quantity
  - Fuel flow rate (lbs/hr)
  - Endurance remaining (time)
  - Compare actual burn vs planned
- [ ] **Time hack** - Check TOT if applicable
  - Monitor ETE (Estimated Time En route) to waypoint on TSD
  - Adjust speed if needed to meet TOT
- [ ] **Communication** - Position reports at checkpoints
  - Report position, altitude, next waypoint, ETA
- [ ] **Threat awareness** - Monitor ASE systems
  - RWR (Radar Warning Receiver) - Watch for radar threats
  - LWS (Laser Warning System) - Watch for laser designators
  - CMWS (Missile Warning) - Watch for missile launches
- [ ] **Weather** - Monitor and adapt
  - Visibility changes
  - Wind shifts (affects groundspeed and fuel)
  - Adjust route if weather deteriorates

## Contour Flight Navigation (Low Level)

- [ ] **Altitude** - 80-200 ft AGL
- [ ] **Airspeed** - 80-120 KIAS
- [ ] **Radar altimeter** - Primary altitude reference
  - Set warning bug at minimum safe altitude (e.g., 50 ft)
  - Audio warning if below bug setting
- [ ] **TSD map** - Monitor terrain contours
  - Anticipate rising/falling terrain
  - Identify valleys, ridges, obstacles
- [ ] **Visual flight** - Primary navigation
  - Continuous scan for obstacles (wires, towers, birds)
  - Correlate visual landmarks with TSD position
- [ ] **Crew coordination** - Both crew scan for threats and obstacles
  - CPG focuses on navigation and threats
  - Pilot focuses on aircraft control and obstacles

## Terrain Flight (Very Low Level)

- [ ] **Altitude** - 25-80 ft AGL
- [ ] **Airspeed** - 60-100 KIAS (reduced from contour)
- [ ] **Radar altimeter** - Critical reference
  - Continuously monitor
  - Set warning bug conservatively
- [ ] **Masking terrain** - Utilize hills, trees, buildings
  - Stay behind terrain to avoid detection
  - Plan routes through valleys and low areas
- [ ] **Obstacle avoidance** - Extreme vigilance
  - Power lines (often nearly invisible)
  - Guy wires (radio towers, construction)
  - Trees and vegetation
  - Birds (especially at dawn/dusk)
- [ ] **TSD navigation** - Reduced map scale (1-2 km)
  - Frequent position updates
  - Short legs between waypoints

## Nap-of-Earth (NOE) Navigation

- [ ] **Altitude** - 0-25 ft AGL (treetop level or below)
- [ ] **Airspeed** - 40-80 KIAS (slow for safety)
- [ ] **Flight profile** - Extreme low altitude
  - Hover taxi in very confined areas
  - Quick pop-ups over obstacles
  - Use every bit of terrain for concealment
- [ ] **Radar altimeter** - Monitor but limited usefulness (too close)
- [ ] **Visual scan** - Intensive by both crew
  - Pilot focused on immediate flight path and obstacles
  - CPG assists with navigation and obstacle callouts
- [ ] **TSD map** - Very close scale (0.5-1 km)
  - Frequent position verification
  - Slow groundspeed allows detailed map reading
- [ ] **Crew coordination** - Constant communication
  - CPG: "Wires 12 o'clock, 200 meters"
  - Pilot: "Visual, climbing over"
  - CPG: "Clear to descend, next waypoint right 30 degrees, 500 meters"
- [ ] **Escape routes** - Continuously identify
  - Always have a "back door" planned
  - Know where to go if ambushed or if emergency
- [ ] **Speed and altitude trade-off**
  - Lower/slower = better concealment but less maneuverability
  - Adjust based on threat level and terrain

## Navigation System Cross-Check

- [ ] **GPS position vs visual** - Cross-check frequently
  - Identify visual landmarks (roads, rivers, towns, distinctive terrain)
  - Compare visual position with TSD aircraft symbol
  - If drift detected, investigate GPS satellite count/quality
- [ ] **Heading cross-check** - Multiple sources
  - Magnetic compass vs GPS track (should match if no wind)
  - HSI vs standby compass
  - GPS heading vs visual ground track
- [ ] **Distance/bearing verification** - To known points
  - Use TSD cursor to measure distance to visible landmarks
  - Verify matches visual estimation
- [ ] **Time check** - Compare actual vs planned
  - ETE (Estimated Time En route) to waypoints
  - Ground speed vs planned speed
  - Adjust for winds aloft
- [ ] **Fuel state verification** - Actual vs planned
  - EUFD FUEL page shows actual burn rate
  - Compare to mission fuel plan
  - If burning more than planned, investigate (winds, altitude, power setting)

## Position Update (Manual)

- [ ] **Visual landmark identification** - Find known point
  - Road intersection, bridge, distinctive terrain feature
  - Verify on map/chart and TSD
- [ ] **TSD cursor function** - Update position if GPS degraded
  - Press CURSOR button on TSD
  - Move cursor to known visual landmark
  - Press UPDATE POSITION (if GPS lost or very inaccurate)
  - Use with caution - GPS usually more accurate than visual
- [ ] **GPS satellite status** - Check if position questionable
  - TSD shows satellite count and PDOP
  - If <4 satellites, GPS accuracy degrades
  - If PDOP >6, position accuracy poor
- [ ] **Waypoint verification** - Confirm next waypoint
  - Check waypoint identifier on TSD
  - Verify bearing and range reasonable
  - If wrong waypoint active, manually sequence correct one

## Holding Pattern (If Required)

- [ ] **Holding waypoint** - Select or create
  - Use existing waypoint or create new point at hold location
- [ ] **Holding pattern** - Establish
  - Altitude: As assigned or tactically appropriate
  - Airspeed: 80-100 KIAS (fuel efficient)
  - Pattern: Racetrack or circular
  - Bank angle: 15-20 degrees (gentle turns)
- [ ] **TSD monitoring** - Track position
  - Stay within designated hold area
  - Monitor fuel during hold
- [ ] **Communication** - Report holding
  - Notify command/tower of holding position and fuel state

## Return to Base (RTB) Navigation

- [ ] **Decision to RTB** - Based on
  - Mission complete
  - Fuel state (bingo fuel)
  - Weather deterioration
  - Aircraft malfunction
  - Crew fatigue
- [ ] **RTB waypoint selection** - On TSD
  - Select Home (H) waypoint or base waypoint
  - Press DIRECT-TO (D→) button for direct navigation
  - HSI needle points directly to destination
- [ ] **Route review** - Plan return
  - Use outbound route or select different return route
  - Consider winds (may affect fuel differently on return)
  - Check for any new threats along return route
- [ ] **Fuel check** - Critical before RTB
  - Current fuel quantity (EUFD FUEL page)
  - Distance to base (TSD shows nm/km)
  - Fuel required to destination + reserve (typically 20 min reserve)
  - If fuel marginal, consider closest FARP or alternate
- [ ] **Winds aloft consideration** - Affects fuel and time
  - Headwind = slower groundspeed, more fuel burn, longer time
  - Tailwind = faster groundspeed, less fuel burn, shorter time
  - Check actual groundspeed on TSD vs airspeed on FLT page
- [ ] **ATIS/Weather** - Monitor approach weather
  - VHF/UHF: Tune tower or ATIS frequency
  - Note active runway, winds, altimeter setting, ceiling/visibility
- [ ] **Altitude profile** - Plan descent
  - If high altitude cruise, plan gradual descent to avoid high rates
  - 500 ft/min descent typical
  - Calculate top of descent point (rule of thumb: 3 nm per 1,000 ft)
- [ ] **Approach brief** - Coordinate with crew
  - Active runway and approach direction
  - Pattern altitude (typically 500 ft AGL for military)
  - Expected landing spot (parking area)
- [ ] **Landing checklist** - Prepare to execute when in terminal area

## Lost Procedures (If Uncertain of Position)

- [ ] **Remain calm** - Aviate, navigate, communicate
- [ ] **GPS check** - Verify GPS lock
  - Check satellite count and PDOP on TSD
  - If GPS valid, trust TSD position
- [ ] **Visual landmarks** - Identify any recognizable features
  - Large towns, major highways, rivers, coastlines
  - Mountains or distinctive terrain
- [ ] **TSD map comparison** - Match visuals to map
  - Zoom out on TSD to see wider area
  - Look for pattern of roads, rivers, terrain that matches visual
- [ ] **Communication** - Request help if needed
  - Contact ATC or command: "Request position confirmation"
  - Provide any known information (last known position, heading, time)
- [ ] **Fuel awareness** - Monitor carefully
  - If lost, may need to land at nearest suitable site
  - Do not fly aimlessly and burn fuel
- [ ] **Emergency actions** - If truly lost and low fuel
  - Consider precautionary landing while fuel and daylight available
  - Activate emergency locator if in distress
